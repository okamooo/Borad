# 中級課題：実装ガイド＆ドハマりチェックリスト（【手順6】）

本課題は、PDF資料『おすすめの作成順番』の **「6.記事とコメント一覧を結合して表示(中級課題)」** に対応しています。
初級で学んだ「JavaとDBの連携」を一歩進め、**パフォーマンス最適化（N+1問題の解決）**と**複雑なデータマッピング**を習得することが目的です。

## 1. 実装の重要ポイント

### ① 「1+N」から「1」への劇的な変化
*   **ポイント**: 初級では「記事取得(1) + 記事の数(N)だけコメント取得」を行っていましたが、中級では `LEFT OUTER JOIN` を使い、ネットワークの往復回数を1回に絞ります。
*   **ドキュメント参照**: `docs/01_domain_repository_index.md` の「3. 学習のポイント」にある通り、実務で数万件のデータを扱う際に、この「1回で取る」スキルがエンジニアの評価を分けます。

### ② オブジェクト指向とRDBの「ギャップ」を埋める
*   **ポイント**: DBから返ってくるデータは「平らな表形式」ですが、JavaのDomainは「記事の中にコメントリストがあるネスト構造」です。
*   **マッピングのコツ**: `ResultSetExtractor` を使い、ループの中で「記事IDが変わったときだけ新しい記事を作る」という**重複排除ロジック**が鍵となります。

## 2. よくある詰まりどころ（トラブルシューティング）

ドキュメント（`docs/spring_boot_troubleshooting_30.md`等）に基づいた、本課題特有のドハマりポイントです。

### 🚨 SQLの「カラム名」重複問題
*   **症状**: `rs.getString("name")` と書くと、記事の名前なのかコメントの名前なのか分からず、意図しない値が入る。
*   **対策**: SQL文で `a.name AS a_name` のようにエイリアス（別名）を必ず付け、Java側でもその別名で指定してください。

### 🚨 1件もコメントがない記事が消える
*   **症状**: JOINを使って実装したら、コメントが付いていない記事が画面から消えてしまった。
*   **原因**: `INNER JOIN` を使っているため、結合相手（コメント）がいない行が除外されています。
*   **対策**: `LEFT OUTER JOIN` を使い、「右側（コメント）が無くても左側（記事）は残す」という結合方式を徹底してください。

### 🚨 `NullPointerException` (リストの初期化忘れ)
*   **症状**: 記事にコメントを追加しようとすると `NullPointerException` が発生する。
*   **原因**: `currentArticle.setCommentList(new ArrayList<>())` を忘れている、またはドメインクラス側で初期化されていない。
*   **参照**: `docs/spring_boot_troubleshooting_30.md` の「28. Listの初期化忘れ」に該当します。

### 🚨 `ResultSet` のループが無限に回る / 終わる
*   **症状**: 記事が1つしか表示されない、またはデータが重複して表示される。
*   **思考のヒント**: `while (rs.next())` の中で「今どの記事を処理しているか」を保持する変数（`lastArticleId` など）の更新タイミングを指差し確認しましょう。

## 3. ステップアップのための思考法

*   **「バケツリレー」の可視化**: 
    DB（平らなデータ）→ ResultSet（行ごとのデータ）→ Java（ネストしたArticle）
    この変換プロセスを、`docs/01_domain_repository_index.md` で推奨されているように「図」でイメージできると、複雑な結合も怖くなくなります。

*   **HTMLの再利用性**:
    Java側のロジックをどれだけ変えても、最終的に `List<Article>` の構造が同じであれば、HTML（Thymeleaf）側を1文字も修正せずに動かせます。これが「データ構造を揃える」ことの強力なメリットです。
