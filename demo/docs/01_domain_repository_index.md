# 【指導・レビューガイド】01. ドメイン設計・リポジトリ作成 〜 一覧表示

このフェーズは、受講生が「データベース」「Javaオブジェクト」「HTML画面」という3つの異なる世界を初めてつなぎ合わせる、掲示板開発の「基礎の土台」となるステップです。

## 1. ロジックの組み立て方（未経験者への導き方）

受講生が「どこから手を付ければいいかわからない」と立ち止まったら、以下の「バケツリレーの設計図」を意識させるよう誘導してください。

### ① Domain（ドメイン）：データの「器」を定義する
*   **導き方**: 「DBのテーブルにある情報を、Javaプログラムの中で『一塊のモノ』として扱うための箱を作ろう」と提案します。
*   **思考のポイント**:
    *   `Article`（記事）と `Comment`（コメント）の2つの箱を作る。
    *   **最大のハードル**: `Article` クラスの中に `List<Comment>` を持たせる設計です。「DBではコメントが記事IDを持っているけれど、Javaでは『記事が複数のコメントを持っている』と表現する方が、この後のプログラムが書きやすくなるんだよ」と、オブジェクト指向の利便性を伝えます。

### ② Repository（リポジトリ）：DBとの「窓口」を作る
*   **導き方**: 「作った箱（Domain）に、DBから取ってきたデータを入れる仕組みを作ろう」と誘導します。
*   **思考のポイント**:
    *   `NamedParameterJdbcTemplate` を使い、SQLを実行する。
    *   `ORDER BY id DESC`（最新順）を意識させる。「掲示板は新しい投稿が上にないと使いにくいよね？」と、ユーザー目線の仕様を思い出させます。

### ③ Controller（コントローラー）：処理の「司令塔」を作る
*   **導き方**: 「リポジトリに『データを頂戴』と命令し、届いたデータを画面へ渡す準備をしよう」と伝えます。
*   **実装クラス**: 今回は表示専用として `BbsDisplayController` を作成し、`@GetMapping("/")` にマッピングします。
*   **思考のポイント**:
    *   `index()` メソッドで全記事を取得する。
    *   記事のリストをループで回し、各記事に対応するコメントを1件ずつセットしていく「地道な作業」のイメージを持たせます。

### ④ HTML（Thymeleaf）：ユーザーへの「見せ方」を作る
*   **導き方**: 「Controllerから届いたデータの数だけ、タグを繰り返し描画しよう」と誘導します。
*   **実装ファイル**: `bbs-display.html`
*   **思考 of ポイント**:
    *   `th:each` を使ったループ処理。
    *   「記事のループの中に、コメントのループがある」という入れ子（ネスト）構造が、Javaの `Article` の中に `List<Comment>` がある構造と一致していることを強調します。

## 2. 重点的な注意点（レビューで指摘すべきポイント）

レビュアーとして、以下のコードが書かれているかを厳しく、かつ優しくチェックしてください。

*   **[仕様] 最新順での取得**:
    *   RepositoryのSQLに `ORDER BY id DESC` が入っているか。
*   **[品質] スネークケースとキャメルケースの変換**:
    *   DBの `article_id` が Javaの `articleId` に正しくマッピングされているか（`BeanPropertyRowMapper` の仕組みへの理解）。
*   **[セキュリティ] XSS対策**:
    *   Thymeleafで表示する際に `th:text` を使っているか（`th:utext` などの危険なタグを使っていないか）。

## 3. 学習のポイント（受講生に深く理解させるべきこと）

### 「N+1問題」という罠
受講生が書く「ループ内でSQLを発行するコード」は、データが少ないうちは正常に動きますが、データが増えると劇的に遅くなります。
*   **指導の言葉**: 「今は記事が少ないから大丈夫だけど、実務で1万件の記事があったら、画面を出すたびに1万回もDBに接続しちゃうんだ。これを『N+1問題』と呼んで、エンジニアが最も警戒すべきパフォーマンス上の問題の一つだよ。今回は構造を理解するためにこのままでいいけど、いつか『JOIN』を使って1回で取る方法も学ぼうね。」

## 4. 予想される質問 ＆ レビュアーとしての回答例

### Q. 「なぜわざわざDomainクラスを作るんですか？ Mapとかでデータを渡せば楽じゃないですか？」
> **回答例**
> 「確かに小さなプログラムならMapでも動くけど、大規模な開発だと『何のデータが入っているか』が誰にでも明確にわかることがすごく大事なんだ。Domainクラスという『定義書』があることで、他のプログラマーも、そして未来の自分も、迷わずに安全にコードが書けるようになる。これが『堅牢なコード』の第一歩だよ。」

### Q. 「th:each の二重ループがこんがらがって、うまく表示されません…」
> **回答例**
> 「まずはJava側のデータを想像してみよう。大きな箱（記事）の中に、小さな箱（コメント）が詰まっているイメージだよね？ HTMLも同じで、まず『大きな箱のループ』を書いて、その内側に『中身を取り出すループ』を書くんだ。混乱したら、まずはコメントを表示せずに、記事だけのループを完成させてから一歩ずつ進めるといいよ。」
